Day 1 · Neural Networks

What is a Neural Network?

A computational model inspired by how your brain works

Biological Neuron

Your brain has ~86 billion neurons. Each receives signals from other neurons, processes them, and fires an output signal if the combined signal is strong enough.

Artificial Neuron

Takes numerical inputs, multiplies each by a weight, adds a bias, passes through an activation function, and produces an output.

output = activation( w₁x₁ + w₂x₂ + … + b )
Universal Approximation Theorem

A neural network with enough neurons and hidden layers can approximate any mathematical function. This is why deep learning is so powerful — it's a universal function approximator.

Day 1 · Activation Functions

Activation Functions

The "switch" that decides how much signal passes through — and adds non-linearity

σ(x) Sigmoid
σ(x) = 1 / (1 + e⁻ˣ)

Outputs between 0 and 1. Great for binary classification output layer. Suffers from vanishing gradients in deep networks.

Output: (0, 1) Binary Classification Vanishing Gradient
Why Do We Need Activation Functions?

Without them, stacking layers is the same as a single linear equation — no matter how deep. Activations let the network learn curves, shapes, and complex patterns instead of just straight lines.

Day 1 · Activation Guide

Choosing the Right Activation Function

The right choice depends on the layer's role and the task type

Hidden Layers
ReLU (Default Choice)

Fast, simple, works well in most networks. Use first.

ReLU(x) = max(0, x)
Leaky ReLU (if ReLU fails)

Fixes "dying ReLU" — neurons that get stuck at 0 and never recover.

LeakyReLU(x) = max(αx, x)
Tanh (for centering data)

Outputs −1 to 1. Better than Sigmoid for hidden layers (zero-centered).

Output Layer
Softmax — Multi-class Classification

Converts outputs to probabilities that sum to 1. Use with Categorical Cross-Entropy.

σ(xᵢ) = eˣⁱ / Σ eˣʲ
Sigmoid — Binary Classification

Single output between 0–1 = probability of class 1. Use with Binary Cross-Entropy.

Linear (none) — Regression

No activation = raw number output. Use with MSE or MAE loss.

Golden Rule: Start with ReLU in hidden layers. Add Softmax/Sigmoid on output based on task. Switch to Leaky ReLU or ELU only if you see problems.

Day 2 · Loss Functions

The Loss / Cost Function

"How wrong are we?" — the compass that guides training

Simple Example

Actual salary (y) = $20K
Random prediction (ŷ) = $15K
Loss = y − ŷ = $5K error

Loss = (y − ŷ)² = 25

We square it to: (1) make all errors positive, (2) penalize big errors more, (3) make differentiation smooth.

Two Types of Problems
Regression

Predict a number
→ House price, salary, temperature

Use: MSE, MAE, Huber

Classification

Predict a category
→ Spam/not spam, dog/cat

Use: Binary CE, Categorical CE

How the Model Learns
1
Compute loss — measure error
2
Compute gradient — which direction to adjust?
3
Update weights — w = w − α(∂c/∂w)
4
Repeat until loss is minimized

Day 2 · Data Preprocessing

Normalization

Putting all features on the same scale — essential for gradient descent to work properly

Without Normalization

Dataset: [10, 12, 11, 13, 1000]

10
12
1000
dominates!

The outlier (1000) dominates all other values. Gradient descent goes haywire.

1. Min-Max Normalization
x' = (x − x_min) / (x_max − x_min)

Scales all values to [0, 1]. Simple and effective. Sensitive to outliers.

Example: x=12, min=10, max=1000 → x' = 0.002
2. Z-Score Standardization
x' = (x − mean) / std

Transforms to mean=0, std=1. More robust to outliers. Standard choice for most neural networks.

Example: x=12, mean=209.2, std=441.6 → x' ≈ −0.45
Why It Matters

Without normalization, gradient descent takes huge steps for big features and tiny steps for small ones — like running with one leg longer than the other. Normalization makes training faster, more stable, and more accurate.

What is an Epoch?

One epoch = one complete pass through the entire training dataset. Training typically runs for 10–100+ epochs, gradually reducing loss each time.

Batch Size

Instead of processing all data at once, training splits it into batches. Each batch updates the weights. Smaller batches = noisier but faster; larger = smoother but slower.

Forward Pass

Data enters the input layer and flows forward through every layer. At each neuron:

1. Compute z = Wx + b (weighted sum)
2. Apply activation: a = f(z)
3. Pass a to the next layer

This produces the final prediction ŷ.

Backward Pass (Backpropagation)

After computing loss, the error flows backwards using the chain rule of calculus:

∂C/∂w = ∂C/∂ŷ · ∂ŷ/∂z · ∂z/∂w

Each weight gets its gradient — how much did it contribute to the error? Then we update: w = w − α(∂C/∂w)

Chain Rule explained simply: "The total error is the product of each layer's contribution." Like a blame chain — if the output was wrong, we trace back who contributed how much to that mistake.

Worked Numerical Example (1 neuron, 1 step)
Forward Pass
w = 0.5, x = 2, b = 0.1
z = (0.5 × 2) + 0.1 = 1.1
a = ReLU(1.1) = 1.1  (positive → pass through)
ŷ = 1.1,  y = 0 (true label)
Loss = (ŷ − y)² = (1.1)² = 1.21
Backward Pass (α = 0.01)
∂C/∂ŷ = 2(ŷ−y) = 2 × 1.1 = 2.2
∂ŷ/∂z = 1  (ReLU: 1 if z > 0)
∂z/∂w = x = 2
∂C/∂w = 2.2 × 1 × 2 = 4.4
wnew = 0.5 − 0.01 × 4.4 = 0.456
Day 2 · Training Foundations

Weight Initialization — Where Training Begins

Random is good — but the right kind of random is critical

The Core Problem: If we initialize weights to all zeros, every neuron computes the exact same output. During backprop, they all receive the same gradient and learn the exact same thing (the symmetry problem). If weights are too large, the signals grow out of control and explode. The goal of initialization algorithms is to start weights randomly, but scaled perfectly so that the variance of the data doesn't grow or shrink as it passes through the layers.
Xavier (Glorot) Init
W ~ N(0, 1/n_in)

For Sigmoid / Tanh activations. Keeps variance stable across layers.

He (Kaiming) Init (2015)
W ~ N(0, 2/n_in)

Introduced by Kaiming He (creator of ResNet) specifically for ReLU activations. Because ReLU outputs zero for exactly half of its inputs (all negative numbers), it effectively cuts the variance of the data in half as it passes through the layer. He Init multiplies the variance by 2 (`2/n_in` instead of `1/n_in`) to perfectly offset this loss, keeping the signal strong!

Golden Rule

Using ReLU → He Init
Using Sigmoid/Tanh → Xavier Init
Most modern frameworks do this automatically.

Day 2 · Training Challenges

Vanishing & Exploding Gradients

The hidden enemy of deep networks — why some networks refuse to learn

The Chain Rule Effect: Backpropagation uses the chain rule, which requires multiplying many derivatives together as the error signal travels backward from the output to the first layer. If those derivatives are small fractions (like 0.25 from a Sigmoid), multiplying them repeatedly shrinks the signal exponentially (Vanishing Gradient). If they are consistently > 1, the signal grows exponentially out of control (Exploding Gradient).
Why Sigmoid Causes Vanishing

Sigmoid's derivative max = 0.25. Each layer multiplies by ≤0.25. After 6 layers: 0.25⁶ = 0.0002 — gradient is essentially zero.

Why ReLU Fixes It

ReLU derivative = 1 for all positive values. Gradient flows through unchanged: 1 × 1 × 1 × 1 = 1. No shrinkage!

Modern Anti-Vanishing Toolkit

How modern architectures guarantee stable gradient flow:

ReLU Family: Derivative is 1 (doesn't shrink signal).
He/Xavier Init: Starts training with perfect variance.
Batch Norm: Re-centers & scales data at every layer.
Grad Clipping: Puts a hard absolute cap on gradients to stop explosions.
Residual (Skip) Connections: Creates an additional "fast lane" for gradients to bypass layers entirely (used in ResNet & Transformers).
Day 2 · Optimization

Gradient Descent

Finding the lowest point in the loss landscape — like a ball rolling downhill

The "Toolkit" from the previous slide exists entirely to make Gradient Descent possible. Gradient Descent requires measuring the slope (gradient) of the loss landscape to know which direction to update weights. If gradients vanish (slope becomes 0), the ball stops rolling and training halts. If gradients explode (slope becomes infinite), the ball is launched out of the landscape. The toolkit ensures the landscape remains smooth and slopes remain stable!
The Analogy

Imagine you're blindfolded on a mountain. You feel the slope under your feet and take a step downhill. That's gradient descent — always stepping in the direction of steepest descent.

The Update Rule
w = w − α · (∂C/∂w)
b = b − α · (∂C/∂b)

α = learning rate (step size)  ·  ∂C/∂w = gradient (slope)

Learning Rate Tips

Too small: learns very slowly, gets stuck
Just right: converges smoothly
Too large: overshoots, may diverge
Typical values: 0.001 to 0.01

Day 2 · Optimizers

Optimizers — How Do We Descend?

Different strategies to find the minimum faster and smarter

Gradient Descent (GD)
W = W − α∇L(W)

Uses the entire dataset for each update. Accurate but very slow for large data.

Converges to optimal
Very slow
Stochastic GD (SGD)
W = W − α∇L(Wᵢ)

Updates weights after each single sample. Fast but noisy — can jump around.

Fast updates
Noisy, may not converge
Mini-Batch GD
W = W − α(1/m)Σ∇L(Wᵢ)

The sweet spot — uses small batches (16–256 samples). Best of GD and SGD.

Most commonly used
Adam (Adaptive Moment Estimation)

Combines Momentum (memory of past gradients — smooths out noise) and RMSprop (adapts the learning rate per parameter). The default choice for most deep learning.

mₜ = β₁mₜ₋₁ + (1−β₁)∇L   ← momentum term
vₜ = β₂vₜ₋₁ + (1−β₂)(∇L)²  ← RMSprop term
W = W − α · m̂ₜ / (√v̂ₜ + ε)
Fast convergence
Adaptive per-param LR
Works out of the box
Convergence Speed — Same Task, Different Optimizers

Convergence means the loss stops improving — the optimizer has found a (local) minimum. A faster-converging optimizer reaches that minimum in fewer iterations, saving training time. GD is slow (full-dataset updates), SGD is noisy (jumps around), Adam is fast and stable.

Day 2–3 · Hyperparameters

Hyperparameters

Settings you configure before training — not learned from data. Found by trial & error.

Learning Rate (α)

Step size for each weight update during training.

Typical: 0.001 – 0.01
Too high → overshoots  |  Too low → slow

Batch Size

Samples processed before updating weights once.

Typical: 16, 32, 64, 128
Small → better generalization  |  Large → faster

Epochs

How many times the model sees the full dataset.

Typical: 10, 50, 100
Too many → overfitting  |  Too few → underfitting

Hidden Layers

Depth of the network — more layers = more abstract features.

Typical: 1, 2, 3, 10
More → better features, but risk of overfitting

Neurons/Layer

Width — how many neurons per hidden layer.

Typical: 32, 64, 128, 512
More → more complex, but more computation

Dropout Rate

% of neurons randomly disabled during training — prevents overfitting.

Typical: 0.1, 0.2, 0.5
Higher → more regularization

How to find them? All of these are found by trial and error (or Grid Search, Random Search, Bayesian Optimization). There's no perfect formula — experimentation is the key.

General Formula
Weights = n_prev × n_curr
Biases = n_curr

For each layer: multiply neurons in the previous layer by neurons in the current layer (for weights), then add one bias per neuron in the current layer.

Limitations of ANNs (Fully-Connected)
No spatial awareness
Treats every pixel independently — can't detect shapes or edges.
Parameter explosion
A 256×256 image → 65,536 inputs. First hidden layer alone needs millions of weights.
No translation invariance
A cat shifted 5 pixels looks completely different to a plain ANN.
️ No sequence memory
Can't handle ordered data like text or audio — no notion of "what came before."
What's Next — Why Convolution & CNNs?

ANNs fail on images. Convolutional Neural Networks (CNNs) solve this with:
Filters that scan for local features (edges, curves, textures)
Parameter sharing — one filter works everywhere in the image
Pooling — built-in translation invariance
